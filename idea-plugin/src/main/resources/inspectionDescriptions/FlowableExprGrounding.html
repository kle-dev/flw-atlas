Reports root identifiers in backend expressions that are neither a known engine root object, an indexed process/case variable, nor a bean referenced elsewhere in the project.

Example: ${ordr.total > 10} is reported when only order is known.

Disabled by default: variables can be created at runtime only (script tasks, APIs, listeners), so an unknown root is a hint, never an error. Known-good roots can be allowlisted via Alt+Enter (stored in .idea/flowable-atlas.xml).
